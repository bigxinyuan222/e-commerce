# 乐享购后台管理系统 - 完整业务流程说明

## 二、各模块详细业务流程

### 2.1 数据统计模块

#### 2.1.1 销售统计看板
**流程：**
1. 选择时间范围（今日/昨日/近7天/近30天/自定义）
2. 展示核心指标：
   - 销售额（已支付订单金额汇总，status >= 2）
   - 订单数（已支付订单数量，status >= 2）
   - 客单价（销售额/订单数）
   - 新增用户数（注册时间在范围内）
3. 趋势图表：每日销售额曲线图
4. TOP榜单：热销商品TOP10、高价值订单TOP10

**数据来源：**
- `order` 表：筛选 status IN (1,2)，按 created_at 过滤
- `user` 表：统计新增用户
- `order_item` 表：商品销量排行

#### 2.1.2 商品分析
**流程：**
1. 查看商品维度数据：
   - 浏览量（需埋点支持）
   - 加购次数（cart表统计）
   - 成交转化率（成交数/浏览量）
   - 平均评分（product_review.avg）
2. 低库存预警：sku.stock < 阈值

### 2.2 商品管理模块

#### 2.2.1 商品列表
**流程：**
1. 展示所有商品（支持搜索：商品名/SKU编码/分类/品牌）
2. 筛选条件：状态（上架/下架）、分类、品牌、价格区间
3. 操作按钮：
   - **新增商品** → 跳转商品编辑页
   - **编辑** → 修改商品信息+规格
   - **上下架** → 切换 product.status
   - **删除** → 软删除 product + sku + spec

**页面元素：**
- 表格列：ID、商品图、名称、分类、品牌、原价、SKU数、状态、创建时间、操作
- 批量操作：批量上架/下架/删除

#### 2.2.2 新增/编辑商品
**流程：**
1. **基础信息**：
   - 填写商品名称、描述、主图、轮播图
   - 选择分类、品牌
   - 设置原价（product.price）
   - 设置状态（默认下架）
2. **规格设置**：
   - 添加规格名（如"颜色"、"尺寸"）
   - 为每个规格名添加规格值（如"红色"、"蓝色"）
   - 生成 SKU 组合（笛卡尔积）
3. **SKU配置**：
   - 为每个SKU设置：
     - SKU编码（唯一标识）
     - 售价（sku.price）
     - 库存（sku.stock）
     - SKU图片（可选，覆盖商品主图）
4. **保存**：
   - 先保存 product → 获取 product_id
   - 保存 specification + specification_value → 获取 spec_id/value_id
   - 保存 product_spec 关联关系
   - 保存 sku（包含 price、stock）

**表单验证：**
- 商品名称必填
- 至少有一个规格
- 每个SKU必须有价格和库存
- SKU编码不能重复

#### 2.2.3 商品详情查看
**流程：**
1. 展示商品完整信息（基础信息+规格+SKU列表）
2. 展示关联数据：
   - 评价摘要（review_summary，AI生成）
   - 参与的活动（秒杀/拼团）

---

### 2.3 库存管理模块

#### 2.3.1 库存日志列表
**流程：**
1. 查询条件：
   - 时间范围
   - 出入库类型（1=销售出库 2=采购入库 3=损耗 4=盘盈 5=盘亏）
   - SKU编码/商品名称
2. 展示字段：
   - 日志ID、SKU编码、商品名称、变动数量、变动前库存、变动后库存
   - 类型、关联订单号（销售出库时）、操作人、创建时间

**数据来源：** `inventory_log` 表

#### 2.3.2 手动调整库存
**流程：**
1. 选择调整类型：采购入库/损耗/盘盈/盘亏
2. 输入SKU编码或搜索商品
3. 填写调整数量（正数增加，负数减少）
4. 填写备注（原因说明）
5. 提交 → 创建 inventory_log + 更新 sku.stock

**权限控制：** 仅商品运营可操作

---

### 2.4 评价管理模块

#### 2.4.1 评价列表
**流程：**
1. 查询条件：
   - 商品名称/SKU编码
   - 评价类型（好评/差评）
   - 状态（显示/隐藏）
   - 时间范围
2. 展示字段：
   - 评价ID、用户昵称头像、商品名称、评价内容（文字+图片）
   - 评分、点赞数、回复数、状态、创建时间
3. 操作按钮：
   - **查看详情** → 展开所有用户回复 + 管理员回复
   - **隐藏/显示** → 切换 product_review.status
   - **回复评价** → 填写 admin_reply（一条评价只能回复一次）
   - **删除回复** → 软删除 review_reply（针对用户间的回复）

**特殊显示规则：**
- 管理员回复（admin_reply）在评价下方高亮显示
- 用户回复（review_reply）以评论列表形式展示
- 已删除的回复显示"该回复已被管理员删除"

#### 2.4.2 评价详情弹窗
**流程：**
1. 左侧：评价原文（含图片预览）
2. 中间：管理员回复区域（如有）
   - 显示回复内容、回复时间
   - 编辑/删除按钮
3. 右侧：用户回复列表
   - 每条回复显示：用户头像昵称、回复内容、点赞数、时间
   - 删除按钮（软删除，仅管理员可见）
4. 底部：点赞记录（哪些用户点了赞）

#### 2.4.3 AI评价摘要审核
**流程：**
1. 查看待审核摘要列表（review_summary.status=0）
2. 展示内容：
   - 商品名称
   - AI生成的摘要文本
   - 基于的评价数量
   - 生成时间
3. 操作：
   - **通过** → status=1，前端展示
   - **拒绝** → 删除该摘要，重新触发AI生成
   - **编辑** → 人工修改后发布

### 2.5 优惠券管理模块

#### 2.5.1 优惠券模板列表
**流程：**
1. 展示所有优惠券模板
2. 筛选条件：状态（启用/禁用）、类型（满减券/抵扣券）
3. 展示字段：
   - 券ID、券名称、面值、使用门槛、有效期
   - 发放总数、已领取数、剩余数、状态、操作
4. 操作按钮：
   - **新增优惠券** → 跳转编辑页
   - **编辑** → 修改券信息（已发放的不可改面值和门槛）
   - **启用/禁用** → 切换 coupon.status
   - **查看领取记录** → 跳转 user_coupon 列表

#### 2.5.2 新增/编辑优惠券
**流程：**
1. **基本信息**：
   - 券名称（如"满100减20"）
   - 券类型：满减券（face_value）/ 抵扣券（discount）
   - 面值
   - 使用门槛（threshold_amount，0表示无门槛）
2. **适用范围**：
   - 指定分类（多选）
   - 指定商品（多选）
3. **发放规则**：
   - 发放总量（0表示不限）
   - 每人限领数量
   - 有效期类型：
     - 固定日期范围（start_time ~ end_time）
     - 领取后N天有效
4. **保存**：
   - 保存 coupon 主表
   - 如果指定范围，保存 coupon_scope（多条记录）

**表单验证：**
- 满减券和抵扣券面值不能超过门槛
- 发放总量 > 已领取数

#### 2.5.3 用户领取记录
**流程：**
1. 查询条件：
   - 优惠券名称
   - 用户手机号/昵称
   - 状态（未使用/已使用/已过期）
2. 展示字段：
   - 记录ID、券名称、用户信息、领取时间
   - 使用时间、关联订单号、状态
3. 操作：
   - **查看详情** → 查看使用订单详情
   - **作废** → 将未使用的券标记为已过期（特殊情况）

### 2.6 营销活动模块

#### 2.6.1 秒杀活动管理

**活动列表：**
1. 展示所有秒杀活动
2. 筛选：状态（未开始/进行中/已结束/已关闭）
3. 操作：
   - **创建活动** → 跳转编辑页
   - **编辑** → 修改活动信息（进行中的不可改）
   - **关闭** → 提前结束活动
   - **查看商品** → 查看该活动的秒杀商品列表

**创建/编辑秒杀活动：**

1. **活动信息**：
   - 活动名称
   - 活动时间（start_time ~ end_time）
2. **添加秒杀商品**：
   - 选择商品
   - 为该商品的每个SKU设置秒杀价
   - 可选：设置该SKU的库存限制（0=不限制，用sku.stock）
3. **保存**：
   - 保存 seckill_activity
   - 保存 seckill_item（多条）
   - 为每个商品保存 seckill_sku_price（多条，一个商品可能有多个SKU）

**秒杀商品列表：**
- 展示该活动下的所有商品
- 点击商品可查看该商品的SKU价格列表
- 可手动下架某个商品

#### 2.6.2 拼团活动管理

**活动列表：**

1. 展示所有拼团活动
2. 筛选：状态、活动名称
3. 操作：
   - **创建活动** → 跳转编辑页
   - **编辑** → 修改活动信息
   - **查看成团情况** → 查看 group_buy_record 列表

**创建/编辑拼团活动：**
1. **活动信息**：
   - 活动名称
   - 活动有效期（start_time ~ end_time）
   - 团的有效期（validity_hours，从开团起算多久内必须成团）
2. **添加拼团商品**：
   - 选择商品
   - 设置该商品的拼团价（group_price）
   - 设置该商品的成团人数（target_count，如3人团、5人团）
3. **保存**：
   - 保存 group_buy_activity
   - 保存 group_buy_item（多条）

**拼团记录列表：**
1. 查询条件：
   - 活动名称
   - 商品名称
   - 团长手机号
   - 状态（拼团中/已成团/已失败）
2. 展示字段：
   - 团ID、活动名称、商品信息、SKU信息、团长信息
   - 当前人数/目标人数、开团时间、过期时间、状态
3. 操作：
   - **查看详情** → 查看团成员列表 + 订单信息
   - **取消团** → 团失败，自动退款给所有成员

**拼团详情：**
- 左侧：团基本信息 + 进度条
- 右侧：成员列表（头像、昵称、加入时间、是否支付）
- 底部：生成的正式订单列表（已成团后才有）

### 2.7 订单管理模块

#### 2.7.1 订单列表
**流程：**
1. 查询条件：
   - 订单号/用户手机号
   - 订单状态（待支付/拼团中/待发货/待自提/已完成/已取消）
   - 门店（门店店员只能看本门店）
   - 时间范围
2. 展示字段：
   - 订单号、用户信息、商品缩略图+名称
   - 订单金额、实付金额
   - 所属门店、订单状态、创建时间
3. 操作按钮：
   - **查看详情** → 跳转订单详情页
   - **取消订单** → 将 status=0 改为 5（待支付订单才能取消）
   - **发货** → 店员确认商品已出库，status=2→3（仅待发货状态有此按钮）

**订单详情页：**
1. **订单信息**：
   - 订单号、下单时间、支付时间、门店发货时间、用户确认自提时间
   - 用户信息（昵称、手机号）
   - 订单状态流转时间轴
2. **商品信息**：
   - 商品列表（名称、规格、单价、数量、小计）
   - 优惠券信息（如使用）
3. **金额明细**：
   - 商品总价
   - 优惠金额
   - 实付金额
4. **门店信息**：
   - 门店名称、地址、联系电话
5. **操作记录**：
   - 谁在什么时候做了什么操作

#### 2.7.2 订单发货与自提流程
**流程：**
1. **店员发货**：
   - 用户在后台找到订单（status=2 待发货）
   - 点击“发货”按钮
   - 系统更新 order.status=3（待自提），记录 shipped_at
   - 提示“发货成功，请通知用户到店自提”
2. **用户到店自提**：
   - 用户到店报手机号或订单号
   - 店员根据手机号/订单号查找订单
   - 确认订单状态为“待自提”（status=3）
   - 店员找到对应商品交给用户
   - 告知用户在手机上点击“确认自提”
3. **用户确认自提**（前台）：
   - 用户打开APP/小程序，进入订单详情
   - 点击“确认自提”按钮
   - 系统更新 order.status=4（已完成），记录 confirmed_at
   - 提示“自提成功，感谢惠顾”

### 2.8 退货退款模块

#### 2.8.1 退款申请列表
**流程：**
1. 查询条件：
   - 订单号/用户手机号
   - 退款状态（待审核/已通过/已拒绝/已完成）
   - 门店（门店店员只能看本门店）
   - 时间范围
2. 展示字段：
   - 退款ID、订单号、用户信息
   - 退款金额、退款原因、凭证图片
   - 申请时间、审核状态、审核人
3. 操作按钮：
   - **审核** → 跳转审核页
   - **查看详情** → 查看完整退款信息

#### 2.8.2 退款审核流程
**流程：**
1. 查看退款申请详情：
   - 订单信息（商品、金额、下单时间）
   - 用户申请的退款原因
   - 用户上传的凭证图片
2. 审核决策：
   - **通过** → 填写审核意见，status=1
     - 通知用户退回商品
     - 等待门店确认收货
   - **拒绝** → 填写拒绝原因，status=2
     - 通知用户拒绝原因
     - 退款流程结束
3. 审核通过后：
   - 用户送回商品
   - 门店收到商品后确认收货 → 财务打款
   - 创建 refund_payment 记录
   - 更新 refund.status=3（已完成）

**审核页面元素：**
- 左侧：订单详情 + 退款申请信息
- 右侧：审核表单
  - 审核结果（通过/拒绝）单选
  - 审核意见（文本框）
  - 提交按钮

#### 2.8.3 退款记录查询
**流程：**
1. 查看所有已完成的退款
2. 筛选：时间范围、门店、退款金额区间

### 2.9 客服消息模块

#### 2.9.1 会话列表
**流程：**
1. 展示所有客服会话
2. 筛选：
   - 状态（待接入/进行中/已关闭）
   - 用户昵称/手机号
   - 最后消息时间
3. 排序：按最后消息时间倒序（最新消息置顶）
4. 展示字段：
   - 用户头像昵称
   - 最后一条消息预览
   - 未读消息数（红点提示）
   - 会话状态
   - 最后更新时间

#### 2.9.2 聊天窗口
**流程：**
1. 点击会话进入聊天窗口
2. 左侧：用户信息卡片（昵称、手机号）
3. 右侧：聊天记录
   - 用户消息（左对齐）
   - 管理员/AI消息（右对齐）
   - 每条消息显示发送时间
   - AI回复标注"AI助手"标签
5. 底部：消息输入框 + 发送按钮 + 表情/图片上传

**消息类型：**

- 文本消息
- 图片消息

**AI兜底机制：**

- 用户发消息后10分钟无人工回复 → 自动触发AI回复
- AI回复显示"AI助手"标识
- 管理员可随时接管会话

#### 2.9.3 会话操作
**流程：**
1. **接入会话**：待接入状态 → 点击"接入" → status=1（进行中）
2. **关闭会话**：问题解决后 → 点击"关闭" → status=2（已关闭）
3. **转接**：转给其他客服 → 选择接收人 → 会话转移
4. **历史记录**：查看该用户的历史会话记录

### 2.10 门店管理模块

#### 2.10.1 门店总览（门店店员专属）
**流程：**
1. 登录自动识别所属门店（admin.store_id）
2. 展示本门店数据：
   - 今日订单数、今日销售额
   - 待发货订单数（status=2的订单）
   - 待自提订单数（status=3的订单）
   - 待处理退款数
3. 快捷操作：
   - **查看订单** → 本门店订单列表
   - **查看退款** → 本门店退款列表

**数据隔离：**
- 门店店员只能看到 store_id = 自己store_id 的数据
- 超级管理员/商品运营/订单客服可看全部门店

### 2.12 管理员管理模块

#### 2.12.1 管理员列表（仅超级管理员可见）
**流程：**
1. 展示所有管理员（除超级管理员自己）
2. 展示字段：
   - 管理员ID、用户名、姓名、手机号
   - 角色、所属门店、状态、创建时间
3. 操作按钮：
   - **新增管理员** → 跳转编辑页
   - **编辑** → 修改信息
   - **禁用/启用** → 切换 admin.status
   - **删除** → 软删除 admin

#### 2.12.2 新增/编辑管理员
**流程：**
1. **基本信息**：
   - 手机号（登录账号，唯一）
   - 密码（首次设置，后续可重置）
   - 姓名、用户名
2. **角色分配**：
   - 选择角色（商品运营/订单客服/门店店员）
   - 如果选"门店店员"，必须选择所属门店
3. **保存**：创建 admin 记录

**权限控制：**
- 只有超级管理员能访问此模块
- 超级管理员账号不在列表中显示，不可删除

### 2.13 首页管理模块

#### 2.13.1 轮播图管理
**流程：**
1. 展示所有轮播图（按排序权重）
2. 展示字段：
   - ID、标题、图片预览、链接类型
   - 链接目标、排序权重、状态、创建时间
3. 操作：
   - **新增轮播图** → 上传图片 + 设置链接
   - **编辑** → 修改图片/链接/权重
   - **删除** → 移除轮播图
   - **上移/下移** → 调整排序

**链接类型：**

- 无跳转
- 商品详情页（选择商品）
- 活动页（选择秒杀/拼团活动）
- 外部链接（输入URL）

#### 2.13.2 推荐位管理
**流程：**
1. 展示所有推荐位（如"热门推荐"、"新品推荐"）
2. 每个推荐位下展示关联的商品列表
3. 操作：
   - **新增推荐位** → 设置推荐位名称、展示位置
   - **添加商品** → 为推荐位选择商品
   - **移除商品** → 从推荐位删除商品
   - **调整排序** → 商品在推荐位中的展示顺序

## 三、通用交互规范

### 3.1 列表页规范
- **搜索栏**：顶部放置常用筛选项 + 搜索按钮 + 重置按钮
- **操作栏**：搜索栏下方，放置"新增"、"导出"、"批量操作"等按钮
- **表格**：
  - 支持列宽拖拽
  - 支持列显示/隐藏
  - 分页器在右下角（每页10/20/50/100条）
  - 空数据时显示友好提示
- **操作列**：固定在右侧，按钮过多时放入"更多"下拉菜单

### 3.2 表单页规范
- **分步表单**：复杂表单用步骤条（如商品编辑：基础信息→规格设置→SKU配置）
- **表单验证**：
  - 实时验证（失去焦点时）
  - 提交时统一验证
  - 错误提示红色文字显示在字段下方
- **保存按钮**：固定在底部右侧，"取消"在左，"保存"在右
- **草稿功能**：长表单支持保存草稿（如文章编辑）

### 3.3 弹窗规范
- **确认弹窗**：删除、禁用等危险操作必须二次确认
- **详情弹窗**：宽度600~800px，超出高度时内部滚动
- **表单弹窗**：用于简单新增/编辑，复杂场景跳转独立页面

### 3.4 状态标识规范
- **成功/正常**：绿色标签
- **警告/待处理**：橙色标签
- **错误/失败**：红色标签
- **信息/已完成**：蓝色标签
- **默认/未知**：灰色标签

### 3.5 消息提示规范
- **成功**：绿色顶部通知，3秒自动消失
- **失败**：红色顶部通知，需手动关闭
- **警告**：橙色顶部通知，5秒自动消失
- **加载**：全屏Loading遮罩（超过1秒的操作）
